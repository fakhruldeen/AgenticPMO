# Executive Overview & Governance Guide

**AgenticPMO** is an enterprise-grade agentic orchestration framework designed to automate PMBOK® 8th Edition compliance, project risk audit, and governance routing. By combining structured state machines with Large Language Model (LLM) intelligence, it enables organizations to run a continuous, error-free PMO layer with minimal overhead.

---

## 🎯 Executive Value Proposition

Traditional Project Management Offices (PMOs) face high administration costs, delayed risk reports, and inconsistent compliance checks. AgenticPMO solves these challenges through autonomous AI agent cooperation:

| Metric | Traditional PMO | AgenticPMO |
| --- | --- | --- |
| **Audit Latency** | Days or weeks (manual checks) | **Real-time** (on every state change) |
| **Compliance Checking** | Periodic audits (sampling-based) | **100% automated coverage** of all artifacts |
| **Escalation Detection** | Self-reported (prone to delay/bias) | **Programmatic & absolute** validation |
| **Administrative Overhead** | High (gathering details, writing drafts) | **Zero-drafting** (automated parameter injection) |

## 🛡️ PMBOK® 8th Edition Governance Tiers

To safeguard organization resources while maintaining velocity, AgenticPMO implements a four-tier **Governance Classification Matrix**. This model ensures operational changes remain rapid, while significant deviations are immediately locked and escalated.

```
  Operational (T1)       Controlled Change (T2)      Sponsor Approval (T3)
┌───────────────────┐    ┌─────────────────────┐    ┌─────────────────────┐
│ PM Delegated      │    │ Director/Board      │    │ Formal Executive    │
│ Decisions         │    │ Review              │    │ Escalation Triggered│
└───────────────────┘    └─────────────────────┘    └─────────────────────┘
  Budget: <= $100K         Cost Var: 5%-10%           Budget: > $100K
  Cost Var: < 5%           Sched Var: 5-15 days       Cost Var: > 10%
  Sched Var: < 5 days      Contingency: $10K-$50K     Sched Var: > 15 days
  Contingency: < $10K                                 Contingency: > $50K
```

### 1. Operational (Tier 1)
- **Authority**: Delegated entirely to the Project Manager (PM).
- **Triggers**: Small budget ($\le$ $100,000) and minor metric variances (cost variance < 5%, schedule variance < 5 days).
- **System Action**: Automatically generates, audits, and signs off on project artifacts.

### 2. Controlled Change (Tier 2)
- **Authority**: PMO Director / Steering Committee review.
- **Triggers**: Moderate variances (cost variance 5%–10%, schedule variance 5–15 days, or contingency spend $10K–$50K).
- **System Action**: Generates change records and routes logs to the PMO board for standard review.

### 3. Governance Change (Tier 3)
- **Authority**: **Project Sponsor (Executive level)**.
- **Triggers**: Budget exceeds $100,000, cost variance exceeds 10%, schedule variance exceeds 15 days, or contingency spend exceeds $50,000.
- **System Action**: **Blocks automatic pipeline completion**. State is flagged with a `[Governor Alert]`, prompting the client to request formal sponsor sign-off.

### 4. Enterprise Portfolio Change (Tier 4)
- **Authority**: **Executive Sponsor & Portfolio Board**.
- **Triggers**: Strategic adjustments, cross-project budget contention, or enterprise resource conflicts.
- **System Action**: Escalates decision metadata to the enterprise portfolio layer, pausing execution until strategic alignment is confirmed.

## 🔒 Automated Quality Gates

Every project artifact (e.g. project charters, governance frameworks, context registers) generated by the agent network must pass through a strict **Compliance Quality Gate**:

1. **Syntactic Check**: The system ensures all required template placeholders (defined by the `pmoskills` SDK database) are successfully populated.
2. **Qualitative Verification**: When LLMs are active, they perform internal consistency checks to verify that cost parameters align with scope descriptions.
3. **Audit Signature**: Upon passing, the system injects a metadata signature block into the markdown document:
   ```yaml
   ---
   doc_id: A05
   status: Approved
   authority: pmoskills
   quality_gate: PASS
   ---
   ```

## 🚀 Deployment Strategy & Roadmap

AgenticPMO operates on a dual-engine architecture:
- **Offline Mode**: Uses deterministic regex matching and logic checks. This allows immediate deployment into on-premise environments, local CLI tools, or CI/CD pipelines without third-party network calls.
- **LLM-Enabled Mode**: Connects to advanced models (Gemini-1.5-Pro or GPT-4-Turbo) via API keys to handle natural language interaction, quality gate auditing, and qualitative governance assessments.
